# Админ-панель T-AUTO

## Доступ к админ-панели

1. Перейдите по адресу: `http://localhost:3000/admin/login`
2. Войдите с учетными данными:
   - **Логин:** `admin`
   - **Пароль:** `admin123`

> ⚠️ **Важно:** В продакшене обязательно измените пароль на более безопасный!

## Возможности админ-панели

### 1. Управление услугами (`/admin/services`)
- Добавление новых услуг
- Редактирование существующих услуг
- Удаление услуг
- Поля: название, описание, ссылка, изображение, иконка

### 2. Управление портфолио (`/admin/portfolio`)
- Добавление новых работ
- Редактирование работ
- Удаление работ
- Поля: категория, название, описание, теги, изображение, иконка

### 3. Управление командой (`/admin/team`)
- Добавление сотрудников
- Редактирование информации о сотрудниках
- Удаление сотрудников
- Поля: имя, должность, описание

### 4. Управление отзывами (`/admin/reviews`)
- Добавление новых отзывов
- Редактирование отзывов
- Удаление отзывов
- Поля: имя клиента, рейтинг (1-5), текст отзыва

## Структура данных

Все данные хранятся в JSON файлах в папке `lib/data/`:
- `services.json` - услуги
- `portfolio.json` - портфолио
- `team.json` - команда
- `reviews.json` - отзывы

## API Endpoints

### Административные (требуют аутентификации):
- `GET /api/admin/services` - получить все услуги
- `POST /api/admin/services` - создать услугу
- `PUT /api/admin/services` - обновить услугу
- `DELETE /api/admin/services?id=X` - удалить услугу

Аналогично для `/api/admin/portfolio`, `/api/admin/team`, `/api/admin/reviews`

### Публичные (для отображения на сайте):
- `GET /api/data/services` - получить услуги
- `GET /api/data/portfolio` - получить портфолио
- `GET /api/data/team` - получить команду
- `GET /api/data/reviews` - получить отзывы

## Безопасность

Текущая реализация использует простую аутентификацию через localStorage. Для продакшена рекомендуется:

1. Использовать более безопасную аутентификацию (JWT, сессии)
2. Хранить пароли в зашифрованном виде
3. Добавить защиту от CSRF атак
4. Ограничить доступ по IP адресу
5. Использовать HTTPS

## Изменение пароля

Чтобы изменить пароль, отредактируйте файл:
`app/admin/login/page.tsx`

Найдите строку:
```typescript
if (username === "admin" && password === "admin123") {
```

И измените на ваши учетные данные.

## Загрузка изображений

Изображения должны быть размещены в папке `public/images/` и указаны в формате:
```
/images/название-файла.jpg
```

## Поддержка

При возникновении проблем проверьте:
1. Запущен ли сервер разработки (`npm run dev`)
2. Существуют ли JSON файлы в `lib/data/`
3. Правильно ли указаны пути к изображениям
4. Консоль браузера на наличие ошибок
